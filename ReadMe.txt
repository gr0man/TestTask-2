Параметры запуска:

hash_test -d d:\tmp\test [-g files_number] [-b min_threads_number] [-e max_threads_number]

	-d - путь к директории с файлами (вложенные директории не обрабатываются).
	-g - генерировать файлы в количестве files_number со случайным содержимым. Имена файлов: 1.bin .. files_number.bin. Размеры файлов задаются в программе константами minFileSize и maxFileSize (utilites.h).
	-b - начальное значение количества потоков.
	-e - конечное значение количества потоков. Максимальное количество запущенных потоков не превышает количества файлов вне зависимости от значения max_threads_number.
	
Информация с временами вычсиления выводится в файл c_logFileName (задается в utilites.сpp).
Реализации хэш-функций находятся в utilites.cpp. 
Добавляются хэш-функции в список для тестирования в конструкторе TestRunner (testrunner.cpp).

TestRunner управляет запуском задач, передавая TaskManager'у путь к директории с файлами и требуемое число потоков. TaskManager создает объект QFuture (инкапсулирован в Task) для каждого потока, передавая ему список файлов для обработки.
Требуемое число потоков последовательно инкрементировется от min_threads_number до max_threads_number. Кроме того, вся последовательность повторяется c_maxIterations раз (для набора статистики), полученные данные усредняются. Весь процесс повторяется для каждой функции из списка.
